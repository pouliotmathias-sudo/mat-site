Aujourd'hui, je te partage de la pure value sur le contexte de Claude Code : comment l'améliorer, comment le raffiner, pis où j'en suis rendu dans mon propre setup. C'est du vécu — incluant mes erreurs.

## 1. Où t'ouvres Claude Code (l'erreur que je faisais moi-même)

L'erreur en partant : j'ouvrais le terminal Windows n'importe où, je tapais `claude`, pis c'était parti. Le problème ? Claude Code était **complètement perdu**. Aucune idée d'où il travaillait, d'où aller chercher le contexte, d'où construire les documents.

Résultat concret : des fois il construisait directement sur mon Desktop, des fois dans mon vault Obsidian (mon second cerveau, où tout mon contexte global vit). Des incohérences, des doublons, du contexte éparpillé.

**Le fix est simple : ouvre toujours Claude Code dans le bon dossier** — celui qui contient ton contexte. Pour moi, c'est le vault Obsidian. Ça a l'air banal, pis c'est la fondation de tout le reste.

## 2. Les routines de début et de fin de session

Le gros point sous-estimé : les **routines**.

**Routine de début de session** : va lire mes courriels, relis ce qu'on a fait la dernière session, regarde où on est rendu dans tel projet, pis dis-moi les priorités pour aujourd'hui.

**Routine de fin de session** : fais un résumé de tout ce qu'on a vu, note les apprentissages, l'avancement des projets, mets à jour les documents.

L'effet composé est puissant : chaque session apprend des précédentes. Claude prend des notes en te quittant, les relit en te retrouvant, pis apprend continuellement tes attentes, tes besoins, tes façons de faire. Tu montes en efficacité à chaque session au lieu de repartir de zéro.

## 3. Un audit hebdomadaire automatisé

Un workflow (n8n ou autre) qui part chaque dimanche pis fait un **audit complet de ta structure de contexte** : dossiers en doublon, fichiers perdus, documents inutiles, failles dans l'organisation. Pis la deuxième automatisation qui va avec : corriger ce que l'audit trouve, apprendre des erreurs, améliorer.

## 4. Hiérarchiser et compresser le vieux contexte

Un workflow qui analyse tous tes documents, les hiérarchise — qu'est-ce qui est encore important, qu'est-ce qui est périmé — pis **compresse** ce qui l'est moins. Parce qu'un contexte énorme, c'est pas un avantage : plus t'ajoutes d'information, plus c'est difficile à structurer pis à exploiter. Un contexte compact et dense bat un contexte gonflé.

## 5. Analyser les relations entre tes notes

Mon second cerveau est dans Obsidian, où tu peux créer des liens entre les concepts. L'automatisation : Claude se promène dans le vault pis **crée des liens entre les idées** — ta stratégie go-to-market qui se connecte à tes tactiques de vente, tes notes de lecture qui rejoignent tes projets.

Ça construit un réseau de neurones dans tes propres idées. Pis c'est ça qui fait que l'IA te sort du contenu **créatif et focus** au lieu du générique : elle pense à travers **tes** connexions, pas celles de tout le monde.

## 6. La veille d'opportunités connectée à ton contexte

Un workflow du dimanche qui analyse les news de la semaine — les grands journaux tech, Y Combinator, les nouveaux repos GitHub qui trendent — pis qui les **croise avec tes projets** : qu'est-ce qui s'applique à toi, quelles opportunités, quelles solutions proposer. La veille générique ne vaut rien ; la veille filtrée par ton contexte vaut de l'or.

## 7. Les loops d'auto-amélioration

En rafale, les loops qui font que le système s'améliore tout seul :

- **Self-improvement loop** — analyse les erreurs, les mauvaises réponses, les répétitions, les pertes de contexte, pis corrige.
- **Problem tracker** — les priorités, les états, les solutions testées, les causes probables — pour que l'IA apprenne à spotter les problèmes avant qu'ils arrivent.
- **La loop du pourquoi** — la plus puissante : une loop qui te demande « t'as décidé ça — pourquoi ça plutôt que ça ? pourquoi dans cet ordre-là ? ». L'objectif : que l'IA apprenne **comment toi tu réfléchis**, pour ensuite réfléchir comme toi de façon autonome.

## Le contexte, c'est LA priorité absolue

Voici la vraie hiérarchie des utilisateurs d'IA en ce moment :

1. Ceux qui utilisent ChatGPT pour leur liste d'épicerie.
2. Ceux qui commencent avec Claude Code, explorent les skills, pluggent leur Gmail.
3. Ceux qui ont bâti **toute leur infrastructure de contexte** — avec des auto-itérations, une IA connectée à leurs ressources, qui apprend d'eux en continu pis devient de plus en plus autonome.

La différence entre les trois est exponentielle. Le contexte, c'est ce qui transforme un outil générique en machine de guerre personnalisée pour les prochaines années.

## Ton devoir

Demande à ChatGPT : « fais-moi un prompt comme si t'étais un développeur senior pour auditer complètement ma structure de contexte Claude Code ». Donne ce prompt à Claude Code. Pis regarde ce qu'il trouve.

Pis si t'as pas encore de contexte pantoute — commence par le point 1 : ouvre Claude Code à la bonne place.
